【実行環境】


【外部ファイルの設定について】
3つの外部ファイルのうち1つについて設定が必要です。
[]で囲まれた部分に以下のように入力してください。
{
    "TOKEN":"[discord_botのトークン]",
    "api_key":"[challongeのAPI_key]",
    "check_responce":"<bound method Response.json of <Response [200]>>",
    "role_unei":"[大会運営者の役職名]",
    "role_ban":"[大会参加停止者の役職名]"
}
【コマンド使い方】
※注意事項
※コマンド名と引数の間には半角スペースを開けてください。
※{}で囲まれた項目は必須項目、[]で囲まれた項目は任意項目です。
※引数を入力する際に括弧(ex.{},[])は不要です。

(参加者向け)
?join {名前}
{名前}という名称で大会に参加します。
discord特有のスタンプ等を名前に利用した場合、名前が化けるのでご注意ください。
大会参加停止のペナルティ役職を持つ場合参加できません。
同名の参加者がいる場合参加できません。名前を変更してください。
大会参加受付が停止されている場合参加できません。その場合運営者に直接伝えてください。

?delete
参加受付を取り消します。
参加していない場合は意味がありません。
大会参加受付が停止されている場合取り消しできません。その場合運営者に直接伝えてください。

(運営者向け)
?create {大会名}
{大会名}という名称の大会を開催します。

?close
大会の参加受付を停止します。
大会の開始時にも自動で参加受付を停止されるので参加受付を停止しなくとも大会を開始できます。

?open
大会の参加受付を再開します。

?shuffle
組み合わせをシャッフルします。
大会の参加受付を終了する前でも行えますが、大会を開始した場合行えません。

?start
大会を開始し、第1試合の選手を呼び出します。

?call [試合番号]
試合の選手をメンション付きで呼び出し、試合の開始宣言をできます。
引数がない場合、コマンドを打つごとに最初に第1試合、次に第2試合…と呼び出せます。
こうして呼び出した場合、3位決定戦は決勝戦の後に呼び出されます。
引数が試合番号の場合、その試合番号の試合の選手を呼び出します。
引数ありのコマンドは引数なしのコマンドに干渉しません。
3位決定戦は決勝戦の次の試合番号となります。

?finalize
大会を終了し、外部ファイルの値を一部リセットします。

?reset
何らかの理由で正式に大会が終了し無かった場合、外部ファイルの値をリセットできます。
